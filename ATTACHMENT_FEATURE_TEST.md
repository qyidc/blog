# 附件功能测试指南

## 功能概述
已为博客系统添加了完整的附件上传和管理功能，支持在文章中插入附件下载链接。

## 新增功能

### 1. 数据库变更
- 新增 `attachments` 表，用于存储附件信息
- 包含字段：id, file_name, file_path, file_size, file_type, upload_at, post_id, download_count, created_at
- 添加相关索引以优化查询性能

### 2. 后端API
- `GET /api/attachments` - 获取附件列表（分页）
- `POST /api/upload-attachment` - 上传附件
- `DELETE /api/attachments/:id` - 删除附件
- `GET /attachments/:id/download` - 下载附件（公共API，自动统计下载次数）

### 3. 前端功能
- 文章编辑页面新增"📎 上传附件"按钮
- 支持上传多种文件类型：.pdf, .doc, .docx, .xls, .xlsx, .ppt, .pptx, .zip, .rar, .txt, .md
- 上传成功后自动在编辑器中插入Markdown格式的下载链接
- 新增"附件管理"页面，可查看、下载、复制链接、删除附件
- 显示附件信息：文件名、大小、上传时间、下载次数

### 4. 文章中的附件使用
上传附件后，会在编辑器中自动插入以下格式的Markdown代码：
```markdown
[📎 文件名.pdf](/attachments/附件ID/download)
```

## 测试步骤

### 1. 数据库迁移
首先需要执行数据库迁移，创建attachments表：
```bash
wrangler d1 execute cf_blog-db --file=./migrations/init.sql
```

### 2. 部署更新
部署更新后的代码到Cloudflare Workers：
```bash
npm run deploy
```

### 3. 功能测试

#### 测试附件上传
1. 登录后台管理界面
2. 进入"文章管理"页面
3. 点击"撰写新文章"或编辑现有文章
4. 在编辑器上方找到"📎 上传附件"按钮
5. 点击按钮，选择一个测试文件（如PDF、Word文档等）
6. 等待上传完成，确认在编辑器中出现了附件链接
7. 保存文章

#### 测试附件下载
1. 在前台查看包含附件的文章
2. 点击附件下载链接
3. 确认文件能够正常下载
4. 检查文件名是否正确

#### 测试附件管理
1. 进入后台"附件管理"页面
2. 查看上传的附件列表
3. 测试"复制代码"功能，确认Markdown代码被复制到剪贴板
4. 测试"下载"功能，确认能够下载附件
5. 测试"删除"功能，确认附件被删除

#### 测试下载统计
1. 多次下载同一个附件
2. 在附件管理页面查看"下载次数"是否增加

## 文件类型支持
支持的附件类型包括：
- 文档类：.pdf, .doc, .docx, .txt, .md
- 表格类：.xls, .xlsx
- 演示类：.ppt, .pptx
- 压缩包：.zip, .rar

## 注意事项
1. 附件文件存储在R2的 `assets/attachments/` 目录下
2. 每个附件都有唯一的ID，通过ID进行下载
3. 删除附件时会同时从R2存储和数据库中删除
4. 下载链接是公开的，不需要登录即可下载
5. 附件大小限制由Cloudflare Workers的限制决定（通常为100MB）

## 故障排除

### 上传失败
- 检查文件类型是否在允许列表中
- 检查文件大小是否超过限制
- 查看浏览器控制台和服务器日志

### 下载失败
- 确认附件ID是否正确
- 检查R2存储中文件是否存在
- 查看服务器日志中的错误信息

### 页面显示异常
- 清除浏览器缓存
- 确认所有JavaScript文件正确加载
- 检查浏览器控制台是否有JavaScript错误
